2020-02-19

（一）回顾目标:

  • 年度目标：

    1. 养成良好的作息习惯；成为一名文采飞扬的技术分享小编。
    2. 成为一名经验丰富的前端架构师
    3. 学会c#
    4. 熟练SQL语句

  • 今日目标：

    1. 优化个人博客加载速度，修改过长的html文件名。
    2. 制定每日复盘模板。
    3. 合理制定月目标。
    4. 制定并尝试自己的每日作息计划。

（二）结果对比

  • 结果和目标一致
  • 结果不如目标
  • 结果超越了目标
  • 结果中出现了目标里没出现的项目
  • 结果中缺失了目标中该出现的项目

（三）叙述过程

客观的陈述今天的行动。

1. 早上7点起床跑步一圈。
2. 10点半左右优化完博客加载速度问题，更改了文章名称。占用了工作时间。
3. 工作时间完成了能源地图网页布局样式的编写。
4. 9:00通过手机便签，制定思考月目标。
5. 晚上9:48开始整理每日复盘模板。

（四）自我剖析

今日认知、想法问题：

1. 网页加载速度问题，还没优化到极致。有了侥幸心理，也没去深度思考自己需要更深入的研究下静态资源加载机制。
2. 晚上萌生了拖延心理。

今日行动、实践问题：

1. 因晚上睡不好，早上爬不起来 - 作息时间还需要调整
3. 并没有按照先易后难的顺序完成工作计划。
4. 月目标的制定，不太确定能不能完成。

（五）总结规律

针对以上问题，以后该怎么做：

1. 坚持按照作息时间行动。
2. 网页加载速度问题。明天晚上抽时间继续研究。
3. 思考好次日工作计划和工作顺序，按计划行事。
4. 先按目标，开始几日的复盘尝试，确定自己的速度，修改月目标。
5. 再有拖延心理萌生时，先让腿动起来。

（六）案例佐证

举例证明总结的规律：

自律，谋定而后动，打破砂锅问到底
